## Architectural decisions (don't re-litigate)

- **Dark-first tokens.** `:root` holds dark theme. `[data-theme="light"]` is the
  opt-in override. Replaces the scaffold's `.dark` class convention.
- **OKLCH master-hue knob.** A single CSS variable `--accent-h` (default `200`) drives
  every accent shade. Reskinning the system is a one-line change.
- **Tailwind v4 + cva.** Translate handoff inline styles to utility classes consuming
  token CSS variables via `@theme inline` in `globals.css`. Hover/focus via `hover:`,
  `focus-visible:` modifiers — no JS state for that.
- **Radix UI for behavior** on overlays (Dialog, Popover, Tooltip, DropdownMenu,
  ContextMenu, HoverCard, Toast) and form primitives (Checkbox, Radio, Switch, Slider,
  Select, Avatar). Ship-It owns styling, Radix owns ARIA + keyboard.
- **Geist self-hosted** via `@fontsource-variable/geist` and `@fontsource-variable/geist-mono`.
  Imported from `globals.css`. Consumers don't depend on Google Fonts CDN.
- **tsup for library builds.** Each lib package has a `tsconfig.build.json`
  (composite=false) referenced from `tsup.config.ts` via top-level `tsconfig:` option.
  Without this the DTS build chokes on `composite: true` in the base tsconfig.
- **vitest-axe** (NOT jest-axe — `jest-axe`'s matcher signature is incompatible with
  Vitest's `expect`).

## Component authoring conventions (every component obeys these)

1. **One folder per component** (or tightly-related family — Button family, Dialog +
   Drawer + Sheet + AlertDialog, Card + StatCard, Avatar + AvatarGroup live together).
2. **Files per folder**: `Component.tsx`, `Component.stories.tsx`, `Component.test.tsx`,
   `index.ts`. Re-export everything from `index.ts`.
3. **`forwardRef` everywhere**. Pass refs through.
4. **`cva` for variant axes** (`variant`, `size`, sometimes `tone`). All conditional
   classes inside the cva declaration — no inline conditional classNames in JSX.
5. **Tailwind utilities consuming token CSS vars**: `bg-accent`, `text-text`,
   `rounded-md`, `border-border`. Never hex values, never primitive utilities.
6. **Hover/focus via Tailwind modifiers**: `hover:`, `focus-visible:`, `disabled:`,
   `data-[state=open]:`. No JS-driven hover state.
7. **Radix wrapping**: when keyboard/ARIA matters (overlays, form controls), wrap a
   Radix primitive. Match its props surface as a passthrough where reasonable; expose
   sub-components via named exports for advanced composition.
8. **`asChild` polymorphism** via `@radix-ui/react-slot` where it makes sense (Button,
   Avatar via Radix Avatar). Note Slot needs `React.Children.only`, so branch the
   render path: asChild = bare passthrough + className; non-asChild = full Component
   render with icon/trailing/loading slots.
9. **Tests** (vitest):
   - render happy path
   - one test per interactive behavior (`userEvent`, never `fireEvent`)
   - controlled-state propagation if applicable
   - axe clean: `expect(await axe(container)).toHaveNoViolations()`
10. **Stories** (Storybook):
    - title `Components/<Group>/<Name>` (Inputs / Display / Overlays)
    - tag `['autodocs']`
    - one story per variant + a composite "matrix" or "states" story
    - argTypes expose every variant prop as a control
11. **Update the public barrel**: add an `export * from './components/<Name>'` to
    `packages/ui/src/index.ts`. Keep groups alphabetical.
12. **Type ergonomics**: when extending `HTMLAttributes<…>`, `Omit` any conflicting
    props (`title` for Card, `children` for Field which uses a render prop).

## Gotchas (don't re-discover these)

- **tsup + composite tsconfig**: causes `error TS6307` in DTS. Solution per-package:
  `tsconfig.build.json` with `composite: false`, referenced via tsup's top-level
  `tsconfig: './tsconfig.build.json'`. Already wired for `tokens`, `icons`, `ui`.
  Replicate for the new `shipit` package in Phase 4.

- **Storybook 8.6 imports**: `@storybook/manager-api` and `@storybook/theming/create`
  moved into `storybook` directly. Use `storybook/internal/manager-api` and
  `storybook/internal/theming/create` (note `internal/`). Already correct in
  `apps/docs/.storybook/manager.ts`.

- **kebab-case regex**: only split letter→Letter (camelCase). Splitting letter→digit
  too breaks `h4` into `h-4`. For tokens like `panel-2` that legitimately need a
  hyphen-then-digit, use a string-literal hyphen in the JS key (`'panel-2': '...'`)
  rather than relying on the regex. See `packages/tokens/src/color.ts` for the pattern.

- **jsdom polyfills** in `packages/ui/src/test/setup.ts`: ResizeObserver,
  IntersectionObserver, matchMedia, scrollIntoView, hasPointerCapture,
  releasePointerCapture. Radix Slider, Select, DropdownMenu need all of these. Replicate
  the same setup file pattern for the new `shipit` package.

- **vitest-axe over jest-axe**: jest-axe's `toHaveNoViolations` matcher uses Jest's
  matcher signature, incompatible with Vitest. Use `vitest-axe` (`expect.extend(...)`
  + import `axe` from `'vitest-axe'`).

- **axe-core color-contrast in jsdom**: throws "canvas not implemented" warning to
  stderr. Harmless — tests still pass. Don't try to "fix" it (would require the heavy
  `canvas` native dep).

- **Slot + multi-children**: Slot requires `React.Children.only`. Conditional rendering
  of icon/trailing/loading slots inside Slot fails. Solution: branch render path on
  `asChild`. See `packages/ui/src/components/Button/Button.tsx`.

- **Empty-package tests**: `vitest run --passWithNoTests` for packages with no test
  files (currently `tokens`).

- **`storybook/test` peer warning**: `@storybook/react` 8.6.18 wants `@storybook/test`
  8.6.18 but 8.6.15 resolved. Cosmetic — doesn't break anything.
