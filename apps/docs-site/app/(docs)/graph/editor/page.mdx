## Accessibility

- Canvas root is `role="application"` with the supplied `aria-label` (default `Graph editor`). One `Tab` brings focus to the canvas; keyboard handlers take over from there.
- `Delete` / `Backspace` removes selected nodes and edges, firing `onNodeDelete` / `onEdgeDelete` for each. Arrow keys nudge selected nodes (Shift = 32px) and fire `onNodeMove`. `Escape` clears the selection and fires `onClearSelection`.
- `âŒ˜Z` / `Ctrl+Z` undoes; `âŒ˜â‡§Z` / `Ctrl+Shift+Z` redoes. `historySize=0` disables the stack entirely if your consumer manages undo externally.
- When `toolbar` is omitted and `onNodeAdd` is provided, the canvas renders a real `<button aria-label="Add node">` in the top-left â€” keyboard-only users can add a node without a pointer.

## Engine

`<GraphEditorCanvas>` uses [React Flow](https://reactflow.dev) (`@xyflow/react`) under the hood. The viewer (`<GraphCanvas>`) uses [Cytoscape.js](https://js.cytoscape.org). They share the `elements[]` shape via a round-trip-tested adapter (`toFlowElements` / `toCytoscapeElements`), and they share token resolution via `@ship-it-ui/graph-tokens`. Consumers who import only the viewer don't pay for React Flow; consumers who import only the editor don't pay for Cytoscape.
